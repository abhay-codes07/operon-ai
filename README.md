## What Operon Does That Nothing Else Can

### SnapBuy вҖ” Autonomous Deal Sniper
> *"Set it. Forget it. Own it."*

Every other price tool sends you an alert. **SnapBuy completes the purchase.** Set a trigger condition in plain English вҖ” "price drops below $899", "GPU back in stock", "visa appointment slot opens". A TinyFish agent watches the URL. The instant the condition is met, it navigates, fills your saved payment details, and finishes checkout. Order confirmation in your inbox before you even see the notification.

```
trigger: "RTX 5090 back in stock at Best Buy"
action:  "add to cart, apply coupons, checkout with saved card"
result:  Order #BB-2847291 confirmed вң“
```

---

### Heartbeat вҖ” Synthetic User Monitoring
> *"Does your product still work? Find out before your users do."*

Define your product's critical flows in plain English. Every 15 minutes, a TinyFish agent acts as a real user going through them: sign up, checkout, create a project, invite a teammate. If anything breaks вҖ” a button doesn't render, a form errors out, a page is blank вҖ” you get an alert with a screenshot of exactly what failed and at which step. Not HTTP 200 checks. **Visual, human-accuracy, UI-level monitoring.**

```
journey: "New User Signup вҶ’ Onboarding вҶ’ First Agent Created"
status:  Step 3 FAILED вҖ” "Stripe form did not render within 8s"
alert:   Sent 19 minutes ago with screenshot evidence
```

---

### PriceWatch вҖ” Price Drop Intelligence
> *"Never pay full price again."*

Add any product from any retailer. Set your target price. TinyFish checks it on your schedule вҖ” hourly, every 6 hours, daily. The moment the price drops below your threshold, you get an instant email + Slack notification with a direct link. Sparkline price history so you can see the trend. Lowest-ever price shown so you know if it's actually a deal.

```
watching: Sony WH-1000XM5 on Amazon  ($279 вҶ’ alert at $249)
history:  $349 вҶ’ $329 вҶ’ $299 вҶ’ $279 вҶ’ ... вҶ’ рҹ”” $249
```

---

### EthicsWatch вҖ” Real-Time ESG & Ethics Intelligence
> *"Know when organizations change their commitments. Before the market does."*

Hedge funds, policymakers, NGOs, and compliance teams monitor organizations for changes in ESG standards, corporate social responsibility commitments, new regulations, governance policies, and supply chain ethics. TinyFish agents scrape corporate CSR pages, SEC EDGAR filings, regulatory body announcements, and UN SDG databases. **Claude analyzes whether a change is materially significant** вҖ” you get a briefing, not a raw diff.

```
monitor: SEC climate disclosure rules
alert:   CRITICAL вҖ” "SEC finalized Scope 1+2 GHG reporting for all large filers"
impact:  "Every public company must update 10-K filings. Non-compliance = material misstatement."
```

---

### Swarm Orchestration вҖ” Fleet at Scale
One command launches parallel agents across multiple URLs simultaneously. Live status grid. Results aggregate automatically when all agents report back.

```bash
POST /api/internal/swarm/launch
{ "targetUrls": ["amazon.com", "ebay.com", "walmart.com", "bestbuy.com"] }
# вҶ’ 4 agents running in parallel right now
```

---

### Sentinel Watchlist вҖ” Semantic Change Detection
Deploy always-on agents monitoring specific URLs on a schedule. Unlike DOM diffing, Claude determines if a change is **semantically meaningful** and generates an intelligence briefing: *"Amazon renamed their Prime tier and added a new upsell step to checkout вҖ” your selector needs updating."*

---

### Agent DNA Transfer вҖ” Behavioral Fingerprinting
After each successful execution, Operon extracts a behavioral fingerprint вҖ” exact action sequences, selector patterns, timing confidence вҖ” and makes it transferable to new agents. Near-instant first-run success. No warmup.

---

### Recon вҖ” Autonomous Security Scanner
Six TinyFish agents simultaneously scan a target domain: admin panel exposure, sensitive file disclosure (`.env`, `.git/config`), login error enumeration, directory listings, unauthenticated API endpoints, missing security headers. Severity-rated findings in minutes.

---

### Pulse вҖ” Competitive Intelligence Engine
Monitor any competitor domain with 3 parallel agents: pricing tiers, hiring signals (8 ML engineers hired = new AI feature in 90 days), feature page changes. Claude synthesizes a battle card: their weaknesses, your advantages, a recommended counter.

---

### DataMesh вҖ” Web-to-JSON Extraction API
Public REST API. Define a schema. Get structured data from any URL.

```bash
curl -X POST https://your-operon.vercel.app/api/v1/extract \
  -H "x-api-key: op_live_xxxx" \
  -d '{"url":"amazon.com/product/...", "schema":{"price":"number","title":"string"}}'
# вҶ’ {"data":{"price":979.99,"title":"Apple iPhone 16 Pro"}, "confidence":0.96}
```

---

### Results Hub вҖ” Unified Output Dashboard
Every execution result in one place вҖ” prices, jobs, extracted data, structured output. Smart preview auto-detects result type. Auto-refreshes every 3 seconds while agents are active.

## Quick Start

### Prerequisites
- Node.js 20+
- Docker
- TinyFish API key вҖ” [tinyfish.ai](https://tinyfish.ai)
- Anthropic API key вҖ” [console.anthropic.com](https://console.anthropic.com) *(optional)*

### 1. Clone & install
```bash
git clone https://github.com/abhay-codes07/operon-ai.git
cd operon-ai
npm install
```

### 2. Configure environment
```bash
cp .env.example .env
# Required: DATABASE_URL, REDIS_URL, TINYFISH_API_KEY, NEXTAUTH_SECRET
```

### 3. Start infrastructure
```bash
docker-compose up -d   # PostgreSQL :5433, Redis :6379
```

### 4. Set up database
```bash
npx prisma migrate deploy
npx prisma db seed
```

### 5. Run
```bash
npm run dev          # Terminal 1 вҖ” Next.js on :3000
npm run worker:dev   # Terminal 2 вҖ” BullMQ worker (required for agent execution)
```

## Deploy to Vercel

### 1. Push to GitHub (already done)
```bash
git push origin main
```

### 2. Create Vercel project
```bash
npm i -g vercel
vercel --prod
# Or: go to vercel.com вҶ’ New Project вҶ’ Import from GitHub
```

### 3. Set environment variables in Vercel dashboard
```
TINYFISH_API_KEY=...
DATABASE_URL=postgresql://...         # Use Neon, Supabase, or Railway
REDIS_URL=redis://...                 # Use Upstash Redis
NEXTAUTH_SECRET=...                   # openssl rand -base64 32
NEXTAUTH_URL=https://your-app.vercel.app
ANTHROPIC_API_KEY=...                 # optional
```

### 4. Deploy worker separately
The BullMQ worker cannot run on Vercel (serverless). Deploy it to:
- **Railway**: `railway up` вҖ” runs the worker process persistently
- **Render**: create a Background Worker service pointing to `worker/index.ts`
- **Fly.io**: `fly deploy` вҖ” Dockerfile included

### 5. Database options (managed PostgreSQL)
| Provider | Free tier | Notes |
|----------|-----------|-------|
| [Neon](https://neon.tech) | вң… 3GB | Serverless, best for Vercel |
| [Supabase](https://supabase.com) | вң… 500MB | Includes auth + storage |
| [Railway](https://railway.app) | вң… 1GB | Also hosts the worker |

### 6. Redis options
| Provider | Free tier | Notes |
|----------|-----------|-------|
| [Upstash](https://upstash.com) | вң… 10K req/day | Serverless Redis, perfect for Vercel |
